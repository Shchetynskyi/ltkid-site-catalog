# UX-FLOW — v1.1 (AS-IS + CONFIRMED CHANGES)

Проєкт: SITE-CATALOG  
Статус: ACTIVE / SOURCE OF TRUTH  
Тип: UX / Product Specification  
Гілка реалізації: feature/catalog-core  

Документ є єдиним джерелом правди для UX:
екрани, переходи, CTA, заборони.
Будь-які зміни — тільки через нову версію цього документа.

---

## 1. Призначення продукту

SITE-CATALOG — публічний сайт-каталог окулярів.

- НЕ інтернет-магазин
- Без кошика, оплати, доставки
- Мета:
  - перегляд асортименту
  - відкриття конкретної моделі
  - перехід до контакту з менеджером або сервісів

---

## 2. Загальні UX-принципи

- Mobile-first
- Один основний сценарій
- Без тупиків
- Мінімум тексту
- Жодних пояснювальних блоків
- Один глобальний CTA

---

## 3. Глобальні елементи

### 3.1 Header (усі екрани)

- Sticky
- Ліва частина:
  - Логотип + текст «Початок»
  - Клік → Home
- Права частина:
  - Кнопка «Запитати менеджера»
  - Веде у зовнішній Messenger
- Меню відсутнє

---

## 4. Екрани та флоу

### 4.1 Home (стартовий екран)

**Роль:** точка входу в каталог

- ЄДИНИЙ стартовий екран
- Без заголовків
- Без пояснювальних текстів

**Структура:**
- 4 клікабельні картки (2×2):
  1. Готові окуляри · Жіночі
  2. Готові окуляри · Чоловічі
  3. Оправи · Жіночі
  4. Оправи · Чоловічі

---

### 4.2 Флоу зі стартового екрану

#### Оправи
- Клік по:
  - «Оправи · Жіночі»
  - «Оправи · Чоловічі»
- → **ОДРАЗУ галерея відповідної категорії**
- Без проміжних екранів

---

#### Готові окуляри
- Клік по:
  - «Готові · Жіночі»
  - «Готові · Чоловічі»
- → **Проміжний екран вибору сценарію**

##### Проміжний екран (готові окуляри)
Містить 2 варіанти:
- «Показати всі»
- «Підібрати за моїм зором»
Цей проміжний екран є окремим UX-екраном вибору сценарію.


**Переходи:**
- «Показати всі» → галерея готових окулярів
- «Підібрати за моїм зором» →
  перехід у **окремий флоу підбору діоптрій**

Примітка:
- Кількість екранів підбору
- логіка звуження діоптрій  
**НЕ описуються в цьому документі**
і мають власний UX-спек.

---

### 4.3 Галерея

**Роль:** перегляд списку моделей

- Список карток моделей
- Кожна картка повністю клікабельна

**Елементи:**
- Sticky-панель під header:
  - «Показати всі»
  - Фільтр ширини оправи
- Лічильник результатів
- Empty state (якщо немає моделей)

**Поведінка:**
- Клік по картці → сторінка моделі
- Scroll позиція зберігається (включно з F5)

---

### 4.4 Картка моделі (в галереї)

**Показує:**
- Preview фото
- MarketingTitle (головний акцент)
- ModelID
- Ціну

**НЕ показує:**
- Матеріал
- Стать
- Розміри
- Бренд
- Технічні деталі

---

### 4.5 Сторінка моделі

**Роль:** прийняття рішення “цікаво / не цікаво”

**Показує:**
- MarketingTitle
- ModelID
- Одне основне фото
- Ціну (велика, під фото)
- Характеристики:
  - Ширина оправи (мм)
  - **Висота оправи (мм)**

**Кнопки сервісів (умовні):**
- «Приміряти на себе» — якщо TryOn = TRUE
- «Подивитись на інших» — якщо AIPreview = TRUE
- Якщо обидва FALSE → блок відсутній

**НЕ показує:**
- Матеріал
- Стать
- Бренд окремо
- Наявність
- Доставка / гарантії / сервіс
- Будь-які пояснювальні тексти

---

### 4.6 Fullscreen фото

- Tap по фото → fullscreen
- Закриття:
  - ✕
  - tap по фону

---

## 5. Навігація та повернення

- Header «Початок» → Home
- Галерея → сторінка моделі
- Сторінка моделі:
  - «Дивитись ще» → повернення у ту саму галерею з фільтрами
  - Кнопка менеджера → зовнішній Messenger

Breadcrumbs відсутні.

---

## 6. CTA

### Головний CTA
- «Запитати менеджера»
- Завжди доступний у header
- Веде за межі сайту

### Другорядні CTA
- TryOn
- AIPreview
- Не є шляхом до покупки
- Можуть бути відсутні

- Змінювати структуру Home
- Додавати нові проміжні екрани
- Міняти тексти кнопок
- Додавати пояснювальні блоки
- Додавати нові CTA
- Змішувати каталог і підбір в один флоу

- екранів

- CTA
- логіки флоу

- нової версії UX-FLOW
